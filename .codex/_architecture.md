## Project Architecture
NOTE: This document is the source of truth for the current website architecture.
If sections below are tagged as (ARCH-MISMATCH), treat them as historical references only.

### Overview
This repository implements the SilicoPharm public website (MVP) with a docs-first structure.

Primary goals:
- Clean, trustworthy docs-style UX
- 4-page MVP: Home / Get Started / Docs / Examples
- Component-based development for Codex iteration
- Easy content operations with Jekyll collections

### Tech Stack
- Static site generator: Jekyll
- Primary style : Google Materialize (`https://materializecss.com/`)
- Content: Markdown (pages + collections)
- Components: Jekyll includes (`_includes/components/*`)
- Styling: CSS tokens + base + component CSS (`assets/css/*`)
- Data-driven navigation: `_data/navigation.yml`, `_data/site.yml`
- Multi lingual support : `index.html`, `index.en.html`, `index.ko.html` and transition button on nav

### Information Architecture (MVP)
Routes:
- `/` Home
- `/get-started/` Get Started
- `/docs/` Docs landing
  - `/docs/getting-started/`
  - `/docs/concepts/`
  - `/docs/condition-spec/`
  - `/docs/api/`
- `/examples/` Examples landing
  - `/examples/target-discovery/` (at least one detailed example)

Navigation:
- Top nav: Home / Get Started / Docs / Examples + CTA ("Try KnockG")

### Content Model
#### Page frontmatter convention
Marketing pages use:
- `layout: marketing`
- `sections: [{type: ComponentName}, ...]`
- component props embedded in page frontmatter (e.g., `hero`, `value_props`)

Docs pages use:
- `layout: docs`
- left sidebar generated by docs structure (minimal implementation acceptable for MVP)

#### Collections (optional for MVP)
- `_docs/` collection for docs pages
- `_examples/` collection for examples pages
(If repo already uses a different structure, keep it consistent and map pages accordingly.)

### Component System
Location:
- `_includes/ui/` for shared UI (HeaderNav, Footer, Container)
- `_includes/components/` for page sections (HeroSection, QuickstartSteps, etc.)

Naming:
- PascalCase filenames: `HeroSection.html`, `QuickstartSteps.html`

Contract:
- Components read props from `page.*` frontmatter keys.
- No hard-coded text inside components except defaults/placeholders.

### Assets
- `assets/img/hero/` for hero backgrounds
- `assets/img/examples/` for example snapshots
- Avoid heavy assets; prioritize SVG/optimized PNG.

### Build & Ops
- Local dev: `bundle exec jekyll serve`
- Build: `bundle exec jekyll build`
- Ensure internal links work and build succeeds before merging.
